Aptitude question bank — seeding runbook

This is the bridge from raw source material (dropped under raw/) to seed-ready JSON in db/seed/items/. It exists because the remaining aptitude content can't be auto-seeded: some sources are images (need hosting) and some have licensing to confirm. Work a source top-to-bottom; nothing reaches the database until you run the seeder yourself.

2. Target shape (what the seeder accepts)

Items are validated by lib/admin/question-bank/item-schema.ts before insert. An aptitude item must have module: "aptitude", a dimension, an options array (each { id, text }), a correctOptionId matching one option id, and a source. Image items add a media object. Copy a starting point from templates/text-item.template.json or templates/figural-item.template.json. 

// text item (numerical | verbal | logical)
{ "module":"aptitude", "dimension":"numerical", "questionText":"…",
  "options":[{"id":"a","text":"…"},{"id":"b","text":"…"}],
  "correctOptionId":"a", "source":"ICAR", "license":"…" }

// figural item (spatial | logical) — option labels are text; the pictures live in media
{ "module":"aptitude", "dimension":"spatial", "questionText":"Which option completes the pattern?",
  "options":[{"id":"a","text":"Option A"},{"id":"b","text":"Option B"}, …],
  "correctOptionId":"c",
  "media":{ "stem":"/aptitude/sandia/mx-001-stem.png",
            "options":{ "a":"/aptitude/sandia/mx-001-a.png", "b":"…" } },
  "source":"SANDIA", "license":"…" }

dimension is free text and becomes its own band in scoring (scoreAptitude groups by it). Use numerical, verbal, logical, and add spatial for figural rotation items.

4. Workflow — text items (numerical / verbal / logical)

  1. Fetch the source page/PDF and save the raw text to raw/<source>/source.txt with the URL + fetch date at the top.
  2. Copy the exact license terms into raw/<source>/LICENSE.txt.
  3. For each item, write one object per templates/text-item.template.json: stem → questionText, choices → options, the answer key → correctOptionId.
  4. Light India-English review (spelling, currency ₹, locale-appropriate names).
  5. Save the array as db/seed/items/<source>-aptitude.json and seed (§6).

5. Workflow — figural / image items

  1. Confirm the source license permits redistribution (§3). If not, stop.
  2. Save each item's pictures to raw/<source>/images/: one stem image + one image per option, named <itemId>-stem.png, <itemId>-a.png, … A page screenshot is not enough — each picture must be cropped to its own file.
  3. Host them where the app can serve them: copy into public/aptitude/<source>/. Next.js serves public/x.png at /x.png, so the seed media URLs are /aptitude/<source>/<itemId>-stem.png etc. (For many items, prefer object storage / CDN instead of the repo.)
  4. Write each item per templates/figural-item.template.json: options[].text are labels ("Option A"); the pictures go in media.stem and media.options[id]; set correctOptionId.
  5. The student UI already renders media.stem (see the aptitude module); option-image rendering may need a small UI follow-up if you use media.options.
  6. Save as db/seed/items/<source>-figural.json and seed (§6).

6. Seed it

pnpm seed:question-bank db/seed/items/<your-file>.json

Idempotent: an item is skipped if a row with the same (source, dimension, questionText, version) already exists, so re-running is safe. Verify the counts it prints, then check the bank: 

select module, source, count(*) from question_bank group by module, source;

7. Before you seed — manual gates

  • ☐ License confirmed for this exact source and recorded in raw/<source>/LICENSE.txt.
  • ☐ No copyrighted/commercial items included (no Raven's, no paid test banks).
  • ☐ Every item has a correctOptionId that matches one of its option ids.
  • ☐ Figural images cropped per-item and hosted; media URLs resolve.
  • ☐ India-English review done.
  • ☐ NTA/CUET content: reuse terms explicitly confirmed, or left out.
Why this is staged, not auto-seeded: figural items need real image assets that have to be cropped and hosted, and several sources need a human license decision. Those are judgment + asset steps, not a scrape — so they live here until a person signs off.
